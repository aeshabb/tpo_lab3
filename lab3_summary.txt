# Лабораторная работа 3: Автоматизированное тестирование UI сложных веб-приложений (Selenium RC)

## Архитектура и методы тестирования: "Под капотом"
В этой лабораторной работе мы не используем современный подход Selenium WebDriver напрямую. Согласно заданию, использован архаичный подход **Selenium RC (Remote Control)** API. 

### Подход: Современный Selenium 4 WebDriver
- **Как работает**: Мы пишем код на Java (`driver.findElement(By.xpath(...)).click()`). WebDriver сериализует команду по протоколу W3C WebDriver во внутренний формат и отправляет ее драйверу. Драйвер непосредственно командует браузером **на глубоком уровне движка** (через CDP или BiDi).
- **Плюсы**: Очень высокая скорость, прямая работа на "нативном" уровне, обход перекрывающихся HTML-элементов и сложного DOM-дерева.

### Подход: Старый Selenium IDE + Selenium RC API
- **Как работает**: Selenium RC (Selenium 1.x) — это устаревшая архитектура. Ранее RC Server запускался как прокси и **внедрял JavaScript-код** (Selenium Core) напрямую в тестируемую веб-страницу.
- **Взаимодействие**: Вместо работы с движком браузера, команды RC (команды из старого IDE типа `selenium.click(locator)`) превращались в JS-события: создавалось синтетическое DOM-событие, которое отправлялось нужному элементу силами самого языка Javascript браузера.
- **Проблемы с современным вебом**: Внедренный JS не умеет кликать по элементу, если он перекрыт прозрачным всплывающим окном, стилем `z-index`, React Overlay-компонентами или находится внутри Shadow DOM.

### Мост в современность: `WebDriverBackedSelenium`
Оригинальный сервер Selenium RC более не совместим с современными Chrome/Firefox. Мы используем класс-адаптер **`WebDriverBackedSelenium`**:
1. Перехватывает старые команды (`selenium.click("xpath=...")`).
2. Транслирует синтаксис локаторов в современные объекты `By`.
3. Эмулирует старое поведение внедряя JS через `JavascriptExecutor`.
**Костыли и хаки**: Интерфейс GitHub блокирует многие скриптовые нажатия. Если команда `click()` через старое API не срабатывает, нам приходилось использовать блок `try...catch` и напрямую выполнять инъекцию `runScript("document.evaluate(...).click();")`.

### Решение проблем с Firefox API Rate Limits (Too many requests)
Старая команда `selenium.type()` в Firefox генерировала лавину запросов к API-автодополнения GitHub на каждый напечатанный символ ввода, что сайт расценивал как DDoS. 
**Решение**:
1. `FirefoxOptions`: (`dom.webdriver.enabled = false`), подмена `User-Agent`.
2. Замена `type()` на `open("/search?q=...")` — браузер перестает "спамить" сервер ложными запросами автодополнения, сразу переходя к результатам.

## Прецеденты тестирования (Use Cases)

UC-01: Поиск репозиториев
Система
Сайт https://github.com
Основное действующее лицо
Пользователь / Гость
Цель
Найти репозиторий по ключевым словам
Триггер
Ввод запроса в строку поиска
Результат
Отображение репозиториев
Основной поток событий:
№ | Действующее лицо | Шаг
1 | Пользователь | Нажимает на кнопку строки поиска в шапке
2 | Пользователь | Вводит запрос «Selenium WebDriver» (или осуществляет переход по URL)
3 | Пользователь | Нажимает Enter
4 | Система | Показывает результаты поиска (results-list)
5 | Пользователь | Нажимает на первый репозиторий в выдаче
6 | Система | Перенаправляет на страницу выбранного репозитория

## Описание набора тестовых сценариев (Test Scenarios)

### Группа 1: Базовый контент и поиск
- **Сценарий: TS-01-01 Успешный поиск репозитория**
  Дано: Пользователь на главной странице GitHub
  Когда: Вводит в поисковую строку "Selenium WebDriver" и подтверждает запрос
  Тогда: Отображается список результатов (`results-list`), после клика на первый элемент происходит роутинг на страницу проекта (адрес не равен `/search`).

- **Сценарий: TS-02-01 Переход в темы и на карточку**
  Дано: Открыта страница `/topics`
  Когда: Пользователь кликает по первой теме в выдаче (тэг `a`)
  Тогда: URL меняется на адрес конкретной темы (содержит `/topics/{название}`).

### Группа 2: Навигация и фильтрация контента
- **Сценарий: TS-03-01 Тренды с использованием фильтрации**
  Дано: Открыта страница трендов платформы
  Когда: Пользователь нажимает на выпадающее меню выбора языка, инжектится JS для обхода Shadow DOM и слоев, выбирается 'Java'
  Тогда: Страница показывает список популярных проектов, URL содержит `java`.

- **Сценарий: TS-05-01 Переход по вкладкам профиля Torvalds**
  Дано: Открыта страница профиля пользователя `torvalds`
  Когда: Осуществляется клик на вкладку `Repositories`
  Тогда: Отображается список всех проектов (`user-repositories-list`) пользователя.

### Группа 3: Служебные и маркетинговые страницы
- **Сценарий: TS-06-01 Ознакомление со страницей подборок**
  Дано: Страница `/explore` загружена
  Когда: Пользователь нажимает на вкладку `Collections`
  Тогда: Отображается список курируемых коллекций с измененным URL.

- **Сценарий: TS-07-01 Просмотр бесплатного тарифа**
  Дано: Страница со сравнением тарифных планов `/pricing`
  Когда: Загружается элемент с текстом `Free`, производится клик на кнопку `Join for free`
  Тогда: Пользователь попадает на страницу создания учетной записи (`/signup` или `/join`).

### Группа 4: Безопасность
- **Сценарий: TS-04-01 Авторизация (Negative State)**
  Дано: Страница авторизации `/login`
  Когда: Пользователь вводит выдуманный логин и пароль и нажимает `Sign in` (кнопку `commit`)
  Тогда: Всплывает блок-уведомление `js-flash-alert` об ошибке аутентификации.
